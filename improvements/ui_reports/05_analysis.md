# UI/UX 개선 보고서: 정책 분석 (Policy Analysis)

정책 분석 페이지는 복잡한 방화벽 규칙 사이의 논리적 오류와 보안 위협을 찾아내는 지능형 워크스페이스입니다. 분석 결과의 명확성을 높이고 사용자 액션을 유도하기 위한 개선 방안을 제안합니다.

---

## 1. 분석 설정 및 실행 (Analysis Configuration)

### 1.1 분석 유형 선택 (Analysis Types)
- **현황**: 드롭다운 형식으로 분석 유형을 선택하고 있어, 각 분석의 차이점이 한눈에 들어오지 않습니다.
- **개선**: 
  - 각 분석 유형을 시각적인 **카드형 선택기(Card Selector)**로 교체하십시오.
  - 각 카드에는 분석 유형을 상징하는 아이콘과 짧은 설명(예: "상/하위 중복 정책 검색")을 포함하여 선택을 돕습니다.
- **실행 버튼**: 분석 실행 버튼은 `btn-primary-gradient`와 함께 `Play` 아이콘을 사용하여 강력한 실행 의지를 표현하십시오.

---

## 2. 분석 결과 요약 (Result Summary)

### 2.1 요약 배너 디자인 고도화
- **현황**: `bg-amber-100` 등 기본 색상을 사용하고 있으며, 정보의 중요도 대비 존재감이 약합니다.
- **개선**: 
  - 중복 발견 시 `bg-ds-tertiary-container/30` 및 `text-ds-tertiary`를 사용하여 "인사이트 발견"을 강조하십시오.
  - 위험 포트 발견 시 `bg-ds-error-container/20` 및 `text-ds-error`를 사용하여 즉각적인 조치가 필요함을 알리십시오.
  - 배경에 미세한 패턴이나 일러스트를 추가하여 "Precision Sentinel"의 프리미엄 느낌을 강화하십시오.

---

## 3. 분석 결과 그리드 (Analysis Grid)

### 3.1 조건부 행 스타일 (Conditional Row Styling)
- **현황**: 중복 정책(상위/하위) 구분 시 배경색(`#e8f4fd`, `#fff8e1`)을 하드코딩하여 사용 중입니다.
- **개선**: 
  - 상위 정책: `bg-ds-primary-container/20`
  - 하위 정책: `bg-ds-secondary-container/30`
  - 위 배경색들을 CSS 변수로 관리하고, 행 좌측에 색상 인디케이터 바를 추가하여 구분을 더 명확히 하십시오.

### 3.2 위험 포트 및 과허용 정책 강조
- **현황**: 위험 포트 정의가 텍스트로만 표시됩니다.
- **개선**: 
  - 위험 포트(Any, 22, 3389 등)는 `ds-error` 색상의 배지 형태로 렌더링하여 보안 관리자가 즉시 인지하게 하십시오.
  - 과허용 범위(Range Size) 수치가 높은 경우 텍스트 크기나 색상을 조절하여 위계감을 부여하십시오.

---

## 4. 정책 영향 분석 (Impact Analysis)

### 4.1 시각적 대비 강화
- **현황**: 이동 대상 정책(Target)과 영향을 받는 정책(Affected)의 구분이 모호합니다.
- **개선**: 
  - **Target Policy**: 굵은 테두리(`border-ds-tertiary`)와 `ds-tertiary/5` 배경색을 적용하십시오.
  - **Affected Policies**: `ds-error/5` 배경색과 함께 "차단됨" 또는 "허용됨" 상태 변화를 아이콘으로 명시하십시오.
