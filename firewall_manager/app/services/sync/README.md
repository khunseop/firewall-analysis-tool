# Synchronization Module (동기화 모듈)

이 모듈은 다양한 제조사의 방화벽 장비로부터 설정 데이터를 수집하고, 이를 시스템 데이터베이스와 동기화하는 핵심 로직을 담당합니다.

## 1. 개요 (Overview)

동기화 모듈은 네트워크 객체, 서비스, 보안 정책 등의 데이터를 정기적으로 또는 수동으로 가져와 최신 상태를 유지합니다. 대량의 데이터를 효율적으로 처리하기 위해 비동기(Async) 처리와 벌크(Bulk) DB 작업을 수행합니다.

## 2. 주요 구성 요소 (Components)

### `tasks.py` (오케스트레이터 및 동기화 로직)
- **`run_sync_all_orchestrator`**: 전체 동기화 프로세스를 관리하는 메인 함수입니다. 연결, 수집, 동기화, 인덱싱의 전 과정을 제어합니다.
- **`sync_data_task`**: 수집된 데이터를 실제 DB와 대량 동기화합니다. 기존 데이터와 비교하여 생성, 수정, 삭제를 결정하며 트랜잭션을 관리합니다.
- **세마포어 관리**: `get_sync_semaphore`를 통해 시스템 자원 고갈을 방지하기 위한 병렬 실행 제한을 수행합니다.

### `collector.py` (데이터 수집기)
- 장비 정보를 바탕으로 적절한 제조사별 Collector 객체를 생성합니다.
- 패스워드 복호화 및 HA Peer IP 처리 등 연결을 위한 전처리를 수행합니다.

### `transform.py` (데이터 변환기)
- 장비로부터 수집된 원시(Raw) 데이터를 시스템 모델 형식에 맞게 정규화합니다.
- Pandas DataFrame을 활용하여 대량의 레코드를 Pydantic 모델로 변환하며, IP 주소 및 포트 번호의 수치화 작업 등을 수행합니다.

## 3. 데이터 흐름 (Data Flow)

1.  **트리거**: 스케줄러 또는 사용자의 요청에 의해 동기화 작업이 시작됩니다.
2.  **준비**: 세마포어를 획득하고 장비에 연결합니다.
3.  **수집 (Export)**: 장비 API/SSH를 통해 객체와 정책 정보를 가져와 DataFrame으로 저장합니다.
4.  **변환 (Transform)**: 제조사별로 상이한 필드명을 표준화하고 데이터를 정제합니다.
5.  **비교 및 동기화 (Sync)**:
    - DB에서 기존 레코드를 조회하여 메모리상에서 새 데이터와 비교합니다.
    - 변경 내역(ChangeLog)을 생성합니다.
    - SQLAlchemy의 `bulk_insert_mappings`, `bulk_update_mappings`를 사용하여 성능을 최적화합니다.
6.  **후처리**: 보안 정책에 대한 전문 검색(Full-text Search) 인덱스를 재구성합니다.

## 4. 핵심 기술 특징

### 병렬 처리 및 자원 관리
- `asyncio.Semaphore`를 사용하여 동시에 실행되는 동기화 작업의 개수를 제한합니다.
- 설정값(`sync_parallel_limit`)에 따라 동적으로 조절 가능합니다.

### Palo Alto 특화 기능 (HA 지원)
- **HA 병렬 수집**: 메인 장비와 HA Peer 장비의 히트 정보(last_hit_date)를 `asyncio.gather`를 사용하여 동시에 수집합니다.
- **히트 정보 병합**: 두 장비 중 더 최신의 히트 시간을 선택하여 정책 사용 이력을 정확하게 관리합니다.

### 데이터 무결성
- 각 데이터 유형별 동기화는 단일 데이터베이스 트랜잭션 블록 내에서 실행되어, 오류 발생 시 부분적인 데이터 오염을 방지합니다.
- 모든 생성, 수정, 삭제 내역은 `change_logs` 테이블에 상세히 기록되어 추적 가능합니다.
